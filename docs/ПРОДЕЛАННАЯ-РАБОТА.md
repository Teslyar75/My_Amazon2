# Что уже сделано в проекте Perry

Документ описывает проделанную работу по части диплома, связанной с **товарами**.  
Проект — маркетплейс по макету Figma «Дипломная работа» (бренд **Perry**, аналог Amazon).  
Логика и сценарии дополнительно взяты из [HomeWork_25.10.2025](https://github.com/Teslyar75/HomeWork_25.10.2025.git) — см. [ИНТЕГРАЦИЯ-HOMEWORK-АДМИНКА.md](./ИНТЕГРАЦИЯ-HOMEWORK-АДМИНКА.md).

---

## 1. Цель твоей части

Backend + витрина + админка для всего, что связано с товаром и покупкой:

- категории (дерево) и товары (каталог, карточка, slug);
- изображения, характеристики, отзывы;
- остатки / статусы / soft-delete (archive);
- корзина (гость + пользователь), заказы, повтор заказа;
- история просмотров (Recently viewed);
- админка категорий / товаров / заказов / пользователей;
- REST API (+ Swagger) для интеграций и диплома.

Стек: **.NET 8**, **ASP.NET Core** (Web API + Razor Pages), **EF Core**, **SQL Server LocalDB**.

## 3. Сущности (Domain)

| Сущность | Назначение | Экран Figma |
|----------|------------|-------------|
| `Category` (+ `Slug`, дерево) | Категории каталога | Admin Category, фильтры, меню |
| `Product` (+ `Slug`, Brand, OldPrice…) | Товар | Product List / Page, Admin Product |
| `ProductImage` | Фото/видео | Галерея |
| `ProductAttribute` | Specs (+ `IsFilterable`) | Таблица характеристик |
| `ProductAboutItem` | About product | Аккордеон |
| `ProductReview` (+ Image, Tag) | Отзывы | Customer Reviews |
| `Cart` / `CartItem` | Корзина (UserId или SessionId) | Cart V2 |
| `Order` / `OrderItem` | Заказ + снимок позиций | Orders / Admin Order |
| `User` / `UserAccess` / `UserRole` | Покупатель и админ | Auth / Admin User |
| `ProductStatus` | Draft / Active / OutOfStock / Archived | бейджи, soft-delete |
| `OrderStatus` | Pending…Cancelled | история / админ |

Файлы: `src/Perry.Domain/Entities/`, `Enums/`.  
У сущностей есть XML-комментарии (`///`).

## 4. База данных (Infrastructure)

| Что | Где / как |
|-----|-----------|
| Контекст | `AppDbContext` |
| СУБД | SQL Server LocalDB → база **`Perry`** |
| Строка подключения | `appsettings.json` в **Api** и **Web** |
| Seed | `DbSeeder` — категории, 8 товаров, картинки, specs, отзывы, slug; backfill slug при старте |
| Конфигурации | `Persistence/Configurations/*.cs` |

### Миграции

| Миграция | Содержание |
|----------|------------|
| `InitialCreate` | Categories, Products, Images, Attributes, About, Reviews, Carts… |
| `AddAdminUsers` | UserRoles, Users, UserAccesses + seed Admin |
| `AddOrders` | Orders, OrderItems |
| `AddProductSlug` | `Products.Slug` (unique) + заполнение из SKU |

Команды:

```bash
dotnet ef migrations add Имя --project src/Perry.Infrastructure --startup-project src/Perry.Web --output-dir Persistence/Migrations
dotnet ef database update --project src/Perry.Infrastructure --startup-project src/Perry.Web
```

Seed: при старте Web/Api вызывается `MigrateAsync` + создание демо-данных, если `Products` пустая.  
Пересоздать с нуля: удали БД `Perry` в LocalDB и снова запусти приложение.

## 6. API (Swagger)

Запуск: `dotnet run --project src/Perry.Api` → `/swagger`.

### Categories

| Метод | URL | Назначение |
|-------|-----|------------|
| GET | `/api/categories` | Дерево корневых + подкатегории |
| GET | `/api/categories/{slug}` | Одна категория |
| POST | `/api/categories` | Создать |
| DELETE | `/api/categories/{id}` | Soft-deactivate (`IsActive = false`) |

### Products

| Метод | URL | Назначение |
|-------|-----|------------|
| GET | `/api/products` | Список + фильтры / sort / page |
| GET | `/api/products/{id}` | Карточка по Guid |
| GET | `/api/products/by-slug/{slug}` | Карточка по slug |
| POST | `/api/products` | Создать |
| DELETE | `/api/products/{id}` | Soft-archive (`Archived`) |

Query списка: `categoryId`, `brand`, `minPrice`, `maxPrice`, `minRating`, `search`, `sort`, `page`, `pageSize`.  
`sort`: `price_desc` (по умолчанию), `price_asc`, `rating_desc`, `newest`.

### Cart

| Метод | URL | Назначение |
|-------|-----|------------|
| GET | `/api/cart?userId=&sessionId=` | Состав + сумма |
| GET | `/api/cart/count?...` | Число позиций |
| POST | `/api/cart/add?...` | Добавить (`ProductId`, `Quantity`) |
| PUT | `/api/cart/quantity?...` | Изменить количество |
| DELETE | `/api/cart/item/{productId}?...` | Убрать |
| POST | `/api/cart/checkout?userId=` | Оформить заказ |

Для API укажи **`userId` или `sessionId`** в query (JWT пока нет — см. «дальше»).

## 7. Витрина и админка (Web)

Auth: session (`AuthSessionMiddleware`), cookie корзины гостя `ds_cart_sid`.  
Выход: `?logout=1`.

### Покупатель / витрина

| URL | Страница | Макет Figma |
|-----|----------|-------------|
| `/` | Главная + Recently viewed | Desktop - Main |
| `/Products` | Каталог: **сайдбар фильтров**, sort, сетка, пагинация | Product List |
| `/Products/Details/{id\|slug}` | PDP: галерея, Add to cart, review, related, viewed | Product Page |
| `/Cart` | Корзина, checkout, viewed | Cart V2 |
| `/Account/Login`, `/Register` | Вход / регистрация (+ merge корзины) | Auth |
| `/Account/Profile` | Профиль: правка, soft-delete, последние заказы | Profile |
| `/Orders`, `/Orders/Details/{id}` | История; **Buy again** | Orders |

### Админка

| URL | Возможности |
|-----|-------------|
| `/Admin/Login` | `Admin` / `Admin` |
| `/Admin` | Stats; создать категорию/товар; deactivate category; archive/restore product |
| `/Admin/Edit/{id}` | Правка товара (+ slug, фото) |
| `/Admin/Orders` | Список заказов, смена статуса |
| `/Admin/Users` | Список пользователей / ролей |

Ключевые UI-файлы:

- `Pages/Shared/_Layout.cshtml` — шапка (Catalog, Account, Cart badge, Admin)
- `Pages/Shared/_ProductCard.cshtml`, `_ViewedProducts.cshtml`
- `wwwroot/css/site.css`, `admin.css`

## 8. Типовые пользовательские сценарии

1. **Гость:** каталог → фильтры → PDP → Add to cart → `/Cart` (бейдж в шапке).  
2. **Просмотры:** открыть 2–3 товара → блок Recently viewed на `/`, `/Products`, `/Cart`, PDP.  
3. **Регистрация / вход:** Register → Login → guest-корзина мержится; Profile.  
4. **Заказ:** Sign in → Checkout → Details → Buy again.  
5. **Отзыв:** на PDP Submit review → пересчёт `AverageRating` / `ReviewCount`.  
6. **Админ:** Login → создать товар → Edit → Archive/Restore → Orders / Users.

## 11. Ребрендинг и UI авторизации (Perry + perry-front)

### Что сделано

1. **Рабочее название диплома — Perry**  
   Решение и проекты переименованы: `DuSoleil.*` → `Perry.*` (`Perry.sln`, namespaces, БД LocalDB `Perry`).  
   Бренд в UI: логотип, title, футер, hero, админка.

2. **Экран входа покупателя** (`/Account/Login`)  
   UI портирован из фронта команды [perry-front](https://github.com/ITSTEP-PERRY/perry-front.git):  
   двухколоночный экран (форма + иллюстрация), шрифт Mulish, цвета `#B8EA48` / `#F4FAFF`, floating-поля, показ пароля.  
   Layout: `_AuthLayout` + `wwwroot/css/auth.css`.  
   Вход принимает логин или email; логика session-auth и merge guest-корзины сохранена.

3. **Экран входа админа** (`/Admin/Login`)  
   Тот же визуал perry-front; учётка по умолчанию `Admin` / `Admin`.

### Файлы UI auth

| Файл | Назначение |
|------|------------|
| `Pages/Shared/_AuthLayout.cshtml` | Layout без Amazon-шапки |
| `Pages/Account/Login.cshtml` | Вход покупателя |
| `Pages/Admin/Login.cshtml` | Вход админа |
| `wwwroot/css/auth.css` | Стили Perry auth |
