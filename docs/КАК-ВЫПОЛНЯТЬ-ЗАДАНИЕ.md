# Как работать с проектом Perry

Практическая инструкция: запуск, разбор кода, сценарии проверки, что можно доделать.  
Читай вместе с [ПРОДЕЛАННАЯ-РАБОТА.md](./ПРОДЕЛАННАЯ-РАБОТА.md) и [ИНТЕГРАЦИЯ-HOMEWORK-АДМИНКА.md](./ИНТЕГРАЦИЯ-HOMEWORK-АДМИНКА.md).

Зона диплома: **товар + покупка** (backend ASP.NET + БД + витрина + админка).

---

## Часть A. Разобраться в уже написанном

### Шаг A1. Открыть solution

1. Открой `d:\Amazon2` в Cursor / Visual Studio.  
2. Открой `Perry.sln`.  
3. Должны быть **4 проекта**: Domain, Infrastructure, Api, Web.

### Шаг A2. Пройти код «сверху вниз»

1. `Web/Program.cs` / `Api/Program.cs` — запуск, seed, middleware  
2. `Infrastructure/DependencyInjection.cs` — БД + сервисы  
3. `Domain/Entities/*.cs` — модель  
4. `Infrastructure/Persistence/AppDbContext.cs` — таблицы  
5. `Infrastructure/Services/*` — бизнес-логика  
6. `Api/Controllers/*` — HTTP  
7. `Web/Pages/*` — UI  

### Шаг A3. Запустить витрину и API

**Витрина:**

```bash
cd d:\Amazon2
dotnet run --project src/Perry.Web
```

| URL | Что смотреть |
|-----|----------------|
| `/` | Главная, Recently viewed (после просмотра товаров) |
| `/Products` | Фильтры слева + сетка |
| `/Products/Details/{slug}` | Карточка (slug или Guid) |
| `/Cart` | Корзина |
| `/Account/Login` | Вход покупателя |
| `/Admin/Login` | Админ: `Admin` / `Admin` |

**API:**

```bash
dotnet run --project src/Perry.Api
```

Swagger → `/swagger` (`categories`, `products`, `cart`).

**Ожидание:** товары есть (seed при старте).

### Шаг A4. База

1. SQL Server Object Explorer / Azure Data Studio.  
2. `(localdb)\mssqllocaldb` → `Perry`.  
3. Таблицы: `Products` (есть `Slug`), `Orders`, `Users`, `Carts`, …

## Часть B. Сверка с Figma

Открой макет «Дипломная работа» и сверь:

| Макет | Уже на витрине / в админке |
|-------|----------------------------|
| Main | hero, категории, deals, best sellers, viewed |
| Product List | фильтры category/brand/price/rating, sort |
| Product Page | галерея, buy box, specs, reviews, related |
| Cart V2 | empty / full / guest hint |
| Auth | Login / Register / Profile |
| Admin * | Dashboard, Edit product, Orders, Users |

Не хватает поля → сущность → миграция → страница/API.

```bash
dotnet ef migrations add Описание --project src/Perry.Infrastructure --startup-project src/Perry.Web --output-dir Persistence/Migrations
dotnet ef database update --project src/Perry.Infrastructure --startup-project src/Perry.Web
```

## Часть D. Сценарии проверки (smoke)

Пройди по порядку после `dotnet run --project src/Perry.Web`:

1. **Фильтры** — `/Products`: бренд, цена, ★4+, Clear.  
2. **Slug** — открой товар, в URL должен быть slug (не только Guid).  
3. **Корзина гостя** — Add to cart → бейдж → `/Cart`.  
4. **Viewed** — 2–3 PDP → блок Recently viewed на главной.  
5. **Register → Login** — корзина сохранилась (merge).  
6. **Checkout** → заказ → **Buy again**.  
7. **Profile** — смени имя; (осторожно) Delete account = soft-delete.  
8. **Admin** — создать товар, Edit, Archive, Restore; Deactivate category; Orders; Users.  
9. **API** — Swagger: `GET /api/products`, `POST /api/cart/add?sessionId=test`.

## Если сломалось

| Проблема | Что проверить |
|----------|----------------|
| Нет БД | LocalDB, `ConnectionStrings` в appsettings |
| Ошибка миграций | `dotnet-ef`, `--project` Infrastructure, `--startup-project` Web |
| Пустой каталог | Seed / статус не Active (Archived скрыт) |
| Порт занят | Другой `dotnet run` или `launchSettings.json` |
| Картинки не грузятся | Нужен интернет (picsum.photos) или свои `/uploads` |
| Корзина «пропала» после логина | Merge только при успешном Login; смотри cookie `ds_cart_sid` |
| API cart ошибка | Передай `userId` **или** `sessionId` |
